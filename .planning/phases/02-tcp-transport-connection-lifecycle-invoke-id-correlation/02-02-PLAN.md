  </read_first>
  <action>
    Extend the argv parser in `main()` to accept `--delay-ms N` and `--close-after N` (each consumes an integer, same style as `--port`/`--fragment`; error to stderr + return 2 on a missing value). Thread both ints through into `runServer` (widen its signature; update the single call site). Do NOT touch the `--selftest` path — `runSelftest()` still returns from `main()` before `runServer`, so neither new flag is parsed there.

    `--delay-ms N` (defer FIRST response, flush LAST — an inline pre-send sleep does NOT reorder because the mock answers frames in receive order; RESEARCH Pitfall 4): add connection-scoped state OUTSIDE the recv loop, per accept — `std::vector<uint8_t> deferred; bool haveDeferred = false; int respCount = 0;`. When a matched command builds its response bytes, increment `respCount`; if `delayMs > 0 && respCount == 1`, stash the bytes in `deferred`/set `haveDeferred` instead of sending; otherwise `sendResponse(...)` immediately. After the inner frame-drain loop for a recv() batch, if `haveDeferred && respCount >= 2`, `usleep(delayMs * 1000)` then `sendResponse(...)` the deferred bytes LAST and clear the flag. On connection close, if a single deferred response is still held (only one request ever arrived), flush it so it is never lost.

    `--close-after N` (deterministic mid-request disconnect): add connection-scoped `int reqCount = 0;`. Where a COMPLETE inbound frame has been parsed (before/at the command switch), increment `reqCount`; if `closeAfter > 0 && reqCount >= closeAfter`, `close(fd)` and break out of the drain/recv loops for this connection WITHOUT sending that response — guaranteeing the client has at least one pending request that must fan out. The outer accept loop continues.

    Keep `--delay-ms`/`--close-after` orthogonal to `--fragment`/`--coalesce` (timing/lifecycle vs segmentation); tests use them independently. Preserve SIGPIPE handling and the existing `LISTENING <port>` readiness line.
  </action>
