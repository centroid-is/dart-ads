  <behavior>
    - decodeSumReadResponse(data, items): reads N×u32 error words, then for each item i: if err_i==0 slice items[i].length bytes and advance cursor by that; if err_i!=0 value is empty Uint8List and cursor advances by 0 (frozen 0-byte-on-failure convention, A1). Returns List<SumResult<Uint8List>>.
    - SUM-04 alignment: a 3-item read where item[1] has err!=0 returns item[0] and item[2] data at CORRECT offsets (item[2] not corrupted by item[1]'s absence).
    - decodeSumWriteResponse(data, n): reads N×u32 error words only, returns List<SumResult<void>>.
    - decodeSumReadWriteResponse(data, n): reads N×(err u32, retLen u32) headers, then slices each block by the RETURNED retLen (failed item retLen==0), NOT the requested length. Returns List<SumResult<Uint8List>>.
    - Over-run: any block whose declared/returned length exceeds remaining bytes throws MalformedFrameException before slicing (T-6-01).
  </behavior>
  <action>
Add the three decoders to `sum_commands.dart`:
- `decodeSumReadResponse(Uint8List data, List<SumReadRequest> items)` — needs the request items because the 0xF080 response carries NO per-item length (per AdsDef.h:70-72; 06-RESEARCH §Pitfall 2). Read the N error words first, then walk the data region using the frozen convention: cursor advances by `items[i].length` for successful items and by 0 for failed items. - `decodeSumReadWriteResponse(Uint8List data, int n)` — N×(err,retLen) headers then data at RETURNED retLen (06-RESEARCH §Wire Layouts C + Anti-Patterns: never slice a READWRITE response by the requested length).
