    - addRoute(netId, host): new NetId → open connection (via injected factory), map, return 0. Same NetId + different host → return 0x0506 (ROUTERERR_PORTALREADYINUSE), leave old route intact. Same NetId + same host → idempotent return 0, no second connection. New NetId + already-used host → return 0 (one-connection-per-NetId; document divergence-from-C++ no-cross-NetId-sharing).
    - removeRoute(netId): unmaps and closes that NetId's connection; getConnection(netId) then null. A different NetId's route is unaffected.
    - setLocalAddress(netId) overwrites localAddr; getLocalAddress() returns it; default is an empty/all-zero AmsNetId.
    - On the first successful connection, if no localAddr was set, derive it as <ip>.1.1 from the connection transport's localAddress (AmsNetId.fromIpv4).

| Boundary | Description |
|----------|-------------|
| caller → router | addRoute host + setLocalAddress NetId are caller-supplied config (trusted; no auto-discovery) |
| router → connection | the router opens sockets to configured hosts only |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-4-01 | Denial of Service | 128-slot port allocator | mitigate | Fixed 128-slot array; `openPort()` returns `0` sentinel on exhaustion (no unbounded allocation, no throw). Higher-level connect() (Plan 04) maps 0 → typed ROUTERERR_NOMOREQUEUES |
| T-4-03 | Tampering | route table host targets | accept | Routes are explicit caller config; auto-discovery is out of scope. Document that route hosts are trusted configuration |
| T-4-04 | Information Disclosure | resolve() missing-route message | accept | The message names only the caller's own unrouted NetId (not secret); it is operational guidance, not a leak |
| T-4-SC | Tampering | package installs | accept | No package installs in this plan (pure-Dart, no pubspec change) |
