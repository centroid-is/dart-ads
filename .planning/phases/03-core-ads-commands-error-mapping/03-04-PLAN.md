Build the `AdsClient` veneer: the idiomatic async Dart API over the six core ADS
commands. Each method encodes the request (existing Phase-1 encoders), sends it via
`AmsConnection.request()`, checks BOTH error levels (AMS errorCode then payload
result) throwing `AdsException`, and returns a typed value (`Uint8List`,

(encoders, decoders, error table, errorCode surfacing) already exists; this is the

    Create `lib/src/client/ads_types.dart` (pure value types):
    - `class AdsStateInfo` with `AdsState adsState`, `int rawAdsState`, `int deviceState`
      (raw ints remain accessible per D-decision).
    - `class DeviceInfo` with `String name`, `int version`, `int revision`, `int build`.

    Create `lib/src/client/ads_client.dart` (async; lives OUTSIDE protocol/):
    - Constructor `AdsClient(AmsConnection connection, {required AmsAddr target, required AmsAddr source})`
      per the locked signature. Store all three. Addressing is stamped by the connection in
      Phase 3; target/source are held as the Phase-4 router seam (document this — do NOT refactor
      AmsConnection addressing this phase).
    - A private helper `_command(int commandId, Uint8List adsPayload, {Duration? timeout})` that
      calls `connection.request`, then: if `resp.errorCode != 0` throw `AdsException.fromCode(resp.errorCode)`;
      else return `resp.payload`. This is the single AMS-level throw site.
    - Named-parameter command methods, each decoding then throwing on `result != 0` via
      `AdsException.fromCode(decoded.result)` (the payload-level throw), then mapping to the typed return:
      - `Future<Uint8List> read({required int indexGroup, required int indexOffset, required int length})`
      - `Future<void> write({required int indexGroup, required int indexOffset, required Uint8List data})`
      - `Future<Uint8List> readWrite({required int indexGroup, required int indexOffset, required int readLength, required Uint8List writeData})` — returns the read bytes directly (D-discretion: yes)
      - `Future<AdsStateInfo> readState()` — maps adsState via AdsState.fromCode, keeps rawAdsState + deviceState
      - `Future<void> writeControl({required AdsState adsState, int deviceState = 0, Uint8List? data})` — passes adsState.code; data defaults to empty (D-decision)
      - `Future<DeviceInfo> readDeviceInfo()`
    Build the raw ADS payloads directly (little-endian, mirroring the encoder bodies in
    commands.dart) since `request()` takes the payload after the 38-byte header. Reference D-01
    (explicit addressing), D-ReadWrite-convenience, and D-WriteControl-data in comments.
  </action>
