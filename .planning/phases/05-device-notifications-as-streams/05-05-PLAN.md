<objective>
Add the `subscribe()` API to `AdsClient`: a single-subscription `Stream<AdsNotification>` whose
`onListen` sends AddDeviceNotification and whose `onCancel` sends DeleteDeviceNotification, with a
state machine handling cancel-during-pending-add. Export the public notification types.

Purpose: this is the user-facing surface for NOTIF-01/02/04. It owns the async lifecycle
orchestration (the pure protocol and the demux map already exist below it), converting
`AdsTransmissionMode` + `Duration` into wire fields and guaranteeing cancel never throws and never
leaks a handle.

Output: edited `ads_client.dart`, barrel exports in `dart_ads.dart`, `test/unit/client/subscribe_test.dart`.
</objective>

  <behavior>
    - subscribe(...) returns a Stream; no Add is sent until `.listen(...)` is called (assert via
      FakeTransport that zero frames were written before listen).
    - On first listen, exactly one AddDeviceNotification (cmd 0x06) is written with the correct
      40-byte payload (mode.code at offset 12, maxDelay/cycleTime at 16/20 in 100ns units).
    - default mode is serverOnChange; cycleTime/maxDelay default Duration.zero -> 0 on the wire.
    - onCancel sends exactly one DeleteDeviceNotification (cmd 0x07) for the acquired handle and the
      cancel Future completes without throwing.
    - Add failure (mock/fake returns error) surfaces via the stream's error, and no Delete leaks.
    - cancel-while-pending-add: if onCancel fires before the Add-response, the just-created handle is
      Deleted once the Add resolves; no unhandled exception, stream ends cleanly.
    - onCancel on a dead connection (deleteNotification throws) is swallowed — cancel still completes.
  </behavior>
  <action>
    Add `Stream<AdsNotification> subscribe({required int indexGroup, required int indexOffset,
    required int length, AdsTransmissionMode mode = AdsTransmissionMode.serverOnChange,
    Duration cycleTime = Duration.zero, Duration maxDelay = Duration.zero, Duration? timeout})`.
    Implement per RESEARCH Pattern 3: create a single-subscription `StreamController<AdsNotification>`,
    hold `int? handle` and `bool cancelled`. In `onListen`: build the Add payload via
    buildAddNotificationPayload (transMode: mode.code, maxDelay100ns: maxDelay.inMicroseconds*10,
    cycleTime100ns: cycleTime.inMicroseconds*10), `await connection.addNotification(payload, ctrl,
    timeout: timeout)`; if `cancelled` became true meanwhile, immediately `_deleteQuietly(h)` and
    return; else store `handle = h`; wrap in try/catch that does `ctrl.addError(e, st)` when not
    closed. In `onCancel`: set `cancelled = true`; if `handle != null` `await _deleteQuietly(handle)`.
    Add a private `Future<void> _deleteQuietly(int h)` that calls `connection.deleteNotification(h,
    buildDeleteNotificationPayload(handle: h))` inside a `try/catch (_) {}` — NEVER rethrow (cancel
    must not throw; the handle is invalidated locally on disconnect anyway — stale-handle rule).
    Do NOT auto-resubscribe (v2 RECON-01): a dead stream stays dead.
  </action>
