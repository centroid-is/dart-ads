  Response addressing INVERTS the request: rTarget = request source (the client), etc.
- AoEHeader command constants: `AoEHeader::ADD_DEVICE_NOTIFICATION` (0x06),
  `AoEHeader::DEL_DEVICE_NOTIFICATION` (0x07), `AoEHeader::DEVICE_NOTIFICATION` (0x08).
- Existing magic groups: `kErrResultGroup = 0xE7700000`, `kErrAmsGroup = 0xE7700001` (declared near top).
- Add request layout (40 bytes): group@0, offset@4, cbLength@8, transMode@12, maxDelay@16, cycleTime@20, 16 reserved@24.
- Add response: result u32=0 + handle u32. Del request: handle u32. Del response: result u32.
- `sendResponse(fd, res, mode, fragmentN, coalesceBuf)` is the send path (used for every response).
