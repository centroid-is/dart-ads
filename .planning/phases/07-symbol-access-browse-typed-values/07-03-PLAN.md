    typeName, comment}`: MAIN.counter DINT @0x4020:0x0 size 4 (ADST_INT32=3), MAIN.flag
    BOOL @0x4020:0x4 size 1 (ADST_BIT=33), MAIN.text STRING(80) @0x4020:0x8 size 81
    (ADST_STRING=30), MAIN.temp LREAL @0x4020:0x60 size 8 (ADST_REAL64=5). Seed the value
    store at each symbol's `{iGroup, iOffs}`. Add per-connection `nextSymHandle` (start 1)
    and a `symHandles` map `handle -> {group, offset}`, both reset per connection.

    Dispatch:
    - 0xF003 (ReadWrite): parse the write payload as the symbol name; strip ONE trailing
      NUL if present (A1); look up by name — miss → ADS result 0x710; hit → allocate
      `handle = nextSymHandle++`, record `handle -> {iGroup,iOffs}`, reply 4-byte LE handle.
    - 0xF005 read (Read): indexOffset = handle → look up symHandles; unknown/released →
      0x710; else return the stored bytes for the symbol's {group,offset}.
    - 0xF005 write (Write): indexOffset = handle → same lookup; unknown → 0x710; else
      overwrite the store at {group,offset}.
    - 0xF006 (Write): indexOffset=0, data = 4-byte LE handle → erase from symHandles
      (idempotent; erasing an unknown handle is a no-op success, matching release semantics).
    - Add `kSymHandleCountGroup` (a sibling magic group, e.g. 0xE7700003u); a Read returns
      `symHandles.size()` as u32 LE for leak assertions.
    Keep the existing notification/sum dispatch untouched.
  </action>
