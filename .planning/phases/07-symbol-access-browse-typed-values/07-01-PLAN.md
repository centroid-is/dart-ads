AdsSymbolEntry byte-exact layout is pinned in 07-RESEARCH.md (30-byte pack(1) header:
six u32 [entryLength, iGroup, iOffs, size, dataType, flags] then three u16 lengths
[nameLength, typeLength, commentLength]; then name\0 type\0 comment\0). ADVANCE BY
entryLength. Read exactly nameLength/typeLength/commentLength bytes; the NUL is a
skip-1 separator. Strings decode Latin-1.
</context>

    with fields: name, typeName, comment (String), indexGroup, indexOffset, size,
    dataTypeId, flags (int). Do NOT store entryLength on the public type — it is a

    Define `List<AdsSymbolInfo> parseSymbolBlob(Uint8List blob, int nSymbols)`. Loop
    nSymbols times over a cursor. Before each header read, require `remaining >= 30`;
    read the 30-byte header via `ByteData.sublistView(blob, cursor)` with `Endian.little`

    `entryLength >= 30 && entryLength <= remaining`. Compute string offsets within the

    `[cursor+30, cursor+entryLength)` (reuse the `_requireBlock`-style subtraction-safe
    guard — never RangeError). Decode each string with `latin1.decode` reading EXACTLY
    the length bytes (do NOT scan for NUL), advancing local offset by length+1 for
    name and type; comment's trailing NUL/padding is absorbed by entryLength. Advance
    the outer cursor by `entryLength` (NEVER by summed field sizes). Break early if the
    cursor reaches blob end before nSymbols. Keep the library NOT re-exported by the barrel.
