<feature>
  <name>CLI value + hex parsing seam</name>
  <files>lib/src/cli/value_parsing.dart, test/cli/value_parsing_test.dart</files>
  <behavior>
  Write the failing tests FIRST (RED), then implement (GREEN), then refactor.
  - `parseHex(String) -> Uint8List`: accepts optional `0x`/`0X` prefix, tolerates
    internal whitespace, requires even nibble count. Cases: "0x0A0B"->[10,11];
    "aa bb"->[170,187]; ""->[] (empty allowed); "0xZZ"->FormatException; "abc"
    (odd length)->FormatException.
  - `formatHex(Uint8List) -> String`: lower-case, `0x`-prefixed, stable
    round-trip with parseHex.
  - `encodeTypedValue(String typeName, String raw, {int? size}) -> Uint8List`:
    dispatches on lower-cased typeName to the value_codec encoder. Cases:
    ("bool","true")->[1]; ("bool","0")->[0]; ("dint","-5")->4 LE bytes;
    ("real","1.5")->4 bytes; ("string","hi",size:8)->8 NUL-padded bytes;
    ("dint","notanint")->FormatException; ("word","70000")->FormatException
    (out of u16 range, never a silent truncation); ("bogus","1")->FormatException
    (unknown type). STRING/WSTRING require `size` or throw FormatException.
  - `decodeTypedValue(String typeName, Uint8List bytes) -> String`: dispatches to
    the value_codec decoder and returns a display string. A bytes buffer SHORTER
    than the type needs throws FormatException (guard BEFORE the codec so no
    RangeError escapes). Round-trips with encodeTypedValue for each scalar.
  Every hostile case asserts a `FormatException` (or a typed subclass), proving
  the T-8-01 no-crash contract.
  </behavior>
  <implementation>
  Delegate all scalar conversion to lib/src/protocol/value_codec.dart (single
  source of truth); this file only parses strings and guards ranges/lengths. Use
  `int.parse`/`double.parse` wrapped so their FormatException propagates, and add
  explicit range checks (u8/u16/u32/i8/i16/i32) that throw FormatException before
  encoding. No dart:io needed here — keep it pure so it is unit-fast.
  </implementation>
</feature>
