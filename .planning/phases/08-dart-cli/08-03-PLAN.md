  - `read --name MAIN.text` (no --type) -> resolves the symbol via browseSymbols

  - `read --name MAIN.counter --raw` -> prints hex (formatHex).
  - `read --group 0x4020 --offset 0 --len 4` -> raw Read, prints hex.
  - `read --name NOPE` -> ADS device error -> exit 1, human-readable name.
  - `read ... --json` -> `{name|group/offset, type, value|hex}` JSON object.
  - group/offset without --len -> UsageException (exit 2).
  </behavior>
  <action>
  Fill ReadCommand.run(): flags `--name`, `--group`, `--offset`, `--len`, `--type`,
  `--raw` (bool), `--json` (bool). Mutually-exclusive name vs group/offset ->
  UsageException if both or neither. By-name path: if `--type` given, use the
  matching typed `readXByName` (map type name -> method via a small switch, reusing
  value_parsing type names); else resolve the symbol from `browseSymbols()` to get
  its `size`+`typeName`, then typed-decode via `decodeTypedValue`, falling back to
  raw hex when the type is unknown. `--raw` forces `readByName(name, size)` +

  int.parse radix handling) and `--len` (required here), call `read(...)`, print
  `formatHex`. `--json` wraps the result. Parse errors -> FormatException (exit 2
  via guarded). Return exitOk.
