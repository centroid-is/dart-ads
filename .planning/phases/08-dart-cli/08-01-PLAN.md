<objective>
Build the CLI backbone that every verb hangs off: the `args`-based CommandRunner
with shared global connection flags, the shared AmsRouter bootstrap/teardown, the
stable exit-code + human-readable ADS-error contract (CLI-08), and seven
registered command STUBS so downstream verb plans each own exactly one command
file with zero shared-file churn.

Purpose: CLI-08 is the interface contract for all other Phase 8 plans. Defining
it first (runner, base command, connection, exit codes, value-parsing seam) means
each verb plan implements against a fixed contract instead of exploring.
Output: `bin/ads.dart`, `lib/src/cli/**`, pubspec `args` + `executables`, and the
exit-code contract integration test.
</objective>

  Create lib/src/cli/connection.dart exposing `Future<AdsSession> connectFromGlobals(ArgResults globals)`
  returning a small `AdsSession` record/class `{AmsRouter router, AdsClient client, Future<void> close()}`.
  Bootstrap steps: parse --timeout ms into a Duration; construct
  `AmsRouter(defaultTimeout: t, connectTimeout: t)`; parse --target (required for
  connection verbs — throw `UsageException` if absent) and --host (required for
  connection verbs). Set the source NetId: if --source given, `AmsNetId.parse` it
  and `router.setLocalAddress(...)`; else if --mode direct and --host is a dotted
  IPv4, derive `AmsNetId.fromIpv4(host)` and setLocalAddress (matches the router's
  <ip>.1.1 rule so tests may omit --source); else if direct with a non-derivable
  host, throw UsageException naming --source. For direct: `router.addRoute(target, host, port:port)`
  then `router.connect(target, amsPort, mode: DirectTarget(deviceHost: host, port: port))`.
  For router mode: `router.connect(target, amsPort, mode: LocalRouterTarget(host: host, port: port))`.
  `close()` must `await router.close()` (tears down every connection + notification
  handles) and be safe to call once from a SIGINT handler and once on normal exit
  (idempotent guard).
  Create lib/src/cli/base_command.dart: `abstract class BaseAdsCommand extends Command<int>`.
  Provide a protected `Future<int> guarded(Future<int> Function() body)` that maps
  thrown errors to exit codes and renders them to stderr, IN THIS ORDER:
  (1) `AdsTimeoutException` / `AdsConnectionException` / `SocketException` -> print
  transport message -> return `exitTransport` (3);
  (2) `AdsException` (incl. AdsRoutingException) -> print
  `"ads error 0x${code.toRadixString(16)} ${adsErrorName(code)}: ${adsErrorText(code)}"`
  (never bare hex alone) -> return `exitAdsError` (1);
  (3) `UsageException` / `ArgumentError` / `FormatException` -> print message ->
  return `exitUsage` (2). Rethrow nothing; unknown -> print + return 1.
  Create the SEVEN command stub files under lib/src/cli/commands/: each a
  `BaseAdsCommand` subclass (`BrowseCommand`, `ReadCommand`, `WriteCommand`,
  `SubscribeCommand`, `PullCommand`, `PushCommand`, `ActionCommand`) with correct
  `name`/`description`, its own `argParser` verb flags declared per 08-CONTEXT
  (so --help is accurate now), and a `run()` that returns `guarded(() async { throw
  UnimplementedError('<verb> lands in a later Phase 8 plan'); })`. Register all
  seven in AdsCliRunner (Task 1). Downstream plans replace ONLY each stub's run()
  body and flag wiring — they do not touch runner.dart.
  </action>

  - Missing --target on a connection verb yields a UsageException -> exit 2.
  </acceptance_criteria>
