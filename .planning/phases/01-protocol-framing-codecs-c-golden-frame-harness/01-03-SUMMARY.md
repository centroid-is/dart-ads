# Phase 01 Plan 03: Mock ADS Server Summary

**C++ mock ADS server with a hand-rolled POSIX accept loop that reuses AmsHeader.h framing structs to answer ReadDeviceInfo byte-accurately, exposes --fragment/--coalesce for Phase-2 TCP reassembly testing, and proves byte-accuracy with a socket-free --selftest gate.**

## Performance

- **Duration:** ~6 min
- **Completed:** 2026-07-03
- **Tasks:** 2
- **Files modified:** 2 (1 created, 1 modified)

## Accomplishments
- `mock_server.cpp`: POSIX `<sys/socket.h>` server binding an ephemeral (`:0`) or `--port N` port, printing `LISTENING <port>` after bind and flushing (Phase-2 readiness handshake).
- Inbound parsing + server-side reassembly via `AmsTcpHeader(ptr)` / `AoEHeader(ptr)` construct-from-bytes; `switch(cmdId)` answers ReadDeviceInfo (0x01) with an AdsLib-framed canned response (result 0, v3.1 build 4024, name "Dart ADS Mock").
- `--fragment N` (N-byte chunked sends, TCP_NODELAY) and `--coalesce` (two frames in one send) transmission modes to drive the Dart FrameAssembler (TEST-04).
- `--selftest` builds the canned response in-process, parses the committed golden hex, and exits 0 on a byte match — an automated byte-accuracy gate with no socket (TEST-01).
- `mock_server` target added to the shared CMake harness; builds green alongside the untouched `dump_golden` target.

## Decisions Made
- A single `buildReadDeviceInfoRes()` builder is shared by both `--selftest` (called with the golden's fixed identities) and the live accept loop (called with the request's echoed identities), guaranteeing the selftest frame is byte-identical to the committed golden.
- Response addressing is passed through un-swapped to match how `dump_golden` generated the golden; realistic target/source swapping is deferred to Phase 2 where the live socket round-trips are wired.
- Reused `dump_golden`'s exact `wrapResponse()` layering (prepend AoEHeader, prepend AmsTcpHeader, patch stateFlags → AMS_RESPONSE) so framing is authoritative by construction.
