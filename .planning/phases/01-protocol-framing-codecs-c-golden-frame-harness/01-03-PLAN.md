  <action>Create test_harness/mock_server.cpp: a small hand-rolled POSIX `<sys/socket.h>` TCP server (avoid AdsLib Sockets.cpp / WinSock — macOS+Linux are both POSIX). Bind an ephemeral port (`:0`) unless `--port N` is given, then print exactly `LISTENING <port>\n` to stdout and flush (this is the Phase-2 readiness handshake — do NOT rely on sleeps). Accept loop: read inbound bytes, use `AmsTcpHeader(ptr)` to get the wrapper length and `AoEHeader(ptr)` to read cmdId + invokeId, then `switch(cmdId)`; for ReadDeviceInfo (0x01) build a canned response reusing `AoEResponseHeader`/AdsLib structs (result u32=0, version/revision/build, 16-byte name) framed with the same prepend order (AoEHeader stateFlags 0x0005, echo the request invokeId, then AmsTcpHeader length = 32 + payload). Support two transmission modes affecting how the response bytes are written to the socket: `--fragment N` sends the response in N-byte chunks with an explicit flush between chunks; `--coalesce` buffers two responses (or the header and body) into a single write. These modes exist to drive the Dart FrameAssembler in Phase 2 (TEST-04). Add a `--selftest` mode that builds the ReadDeviceInfo response in-process, compares its bytes against `test/golden/read_device_info_res.hex` (strip `#` comments/whitespace), prints OK and exits 0 on match / non-zero on mismatch — giving an automated byte-accuracy gate with no socket. The mock's command table is intentionally minimal (ReadDeviceInfo only) and grows in later phases per the locked decision.</action>
